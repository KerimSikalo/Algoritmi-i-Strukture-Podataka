## 🛠️ Implementation Examples

### 📝 **Quick Sort Implementation**
```cpp
// Quick Sort with Lomuto Partition
void quickSort(vector<int>& arr, int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}

int partition(vector<int>& arr, int low, int high) {
    int pivot = arr[high];
    int i = (low - 1);
    
    for (int j = low; j <= high - 1; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    swap(arr[i + 1], arr[high]);
    return (i + 1);
}
```

### 🌳 **Binary Search Tree Implementation**
```cpp
class BST {
    struct Node {
        int data;
        Node* left;
        Node* right;
        
        Node(int val) : data(val), left(nullptr), right(nullptr) {}
    };
    
    Node* root;
    
public:
    BST() : root(nullptr) {}
    
    void insert(int val) {
        root = insertRec(root, val);
    }
    
    Node* insertRec(Node* node, int val) {
        if (node == nullptr) {
            return new Node(val);
        }
        
        if (val < node->data) {
            node->left = insertRec(node->left, val);
        } else if (val > node->data) {
            node->right = insertRec(node->right, val);
        }
        
        return node;
    }
};
```

## 📈 Complexity Analysis

### 🎯 **Big-O Notation Guide**

<div align="center">

| **Notation** | **Name** | **Example Operations** | **Performance** |
|--------------|----------|------------------------|-----------------|
| **O(1)** | Constant | Array access, hash lookup | 🟢 Excellent |
| **O(log n)** | Logarithmic | Binary search, balanced tree ops | 🟢 Very Good |
| **O(n)** | Linear | Linear search, array traversal | 🟡 Good |
| **O(n log n)** | Linearithmic | Merge sort, heap sort | 🟡 Acceptable |
| **O(n²)** | Quadratic | Bubble sort, nested loops | 🔴 Poor |
| **O(2ⁿ)** | Exponential | Recursive Fibonacci | 🔴 Very Poor |

</div>

### 📊 **Space vs Time Trade-offs**
Understanding when to optimize for time vs space complexity:
- **Time-Critical Applications**: Gaming, real-time systems
- **Space-Critical Applications**: Embedded systems, mobile apps
- **Balanced Approach**: Most general-purpose applications
